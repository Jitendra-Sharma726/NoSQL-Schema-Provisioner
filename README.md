# NoSQL-Schema-Provisioner with Terraform

## Project Overview

Developed an Infrastructure as Code (IaC) solution using Terraform to automate the provisioning and configuration of a cloud-native NoSQL database on AWS DynamoDB. The project demonstrates how to define a scalable database architecture by configuring a DynamoDB table, establishing a primary partition key, and enabling on-demand billing for cost-efficient data storage and retrieval.

The solution leverages Terraform's declarative configuration model to provision database resources consistently and reproducibly. By defining the table schema through a partition key, the database can efficiently distribute and retrieve records at scale while maintaining the flexibility of a schemaless NoSQL design.

## Responsibilities

* Designed and provisioned AWS DynamoDB resources using Terraform.
* Configured a DynamoDB table with a primary partition key.
* Implemented on-demand billing using PAY_PER_REQUEST mode.
* Defined key attributes and data types for efficient data access.
* Executed Terraform initialization, planning, and deployment workflows.
* Managed infrastructure state for repeatable deployments.
* Applied Infrastructure as Code best practices for database provisioning.
